# Known Issues — nikki-san

This document lists **known issues and common pitfalls** in the nikki-san blog system.  
It helps AI assistants avoid incorrect debugging approaches.

---

# Hugo Build Errors

## JSON Template Rendering Errors

Example error:

```text
unexpected character '<'
```

Cause:
A JSON template attempted to render HTML.

Typical scenario:
A template renders `.Content` or `.Page.RenderString` inside a JSON output format.

Fix approach:
Check templates under `layouts/` and ensure that JSON templates output valid JSON.

# Hugo Local vs Production Behavior

Some features behave differently between:
- local development
- Cloudflare Pages deployment

Example causes:
- external APIs
- JavaScript-based features
- GitHub-based comment systems

When debugging issues that only appear locally, verify whether the feature depends on network access.

# Theme Override Confusion

The Stack theme may contain multiple files with similar names.

If the same file exists in both the theme and the repository override directory, Hugo uses the override version.

Debugging rule:
Always check whether a file exists under `layouts/` before modifying upstream theme assumptions.

---

# Hugo Modules vs `themes/` Directory Confusion

This project uses **Hugo Modules** for the Stack theme.

Common pitfall:
An AI may tell the user to edit `themes/hugo-theme-stack/...` even though the repository has no local `themes/` directory.

Correct approach:
- confirm whether the project is module-based
- prefer local overrides under `layouts/`, `assets/`, or `static/`
- only reference the upstream theme path as a source example, not as a guaranteed local file path

# Stack Footer Path Confusion

For the current upstream Stack theme, footer files are located under underscore-prefixed paths such as:

```text
layouts/_partials/footer/footer.html
```

not necessarily under:

```text
layouts/partials/footer/footer.html
```

Common pitfall:
An AI assumes the old or wrong path and gives invalid instructions.

Correct approach:
- verify the active upstream theme structure
- verify the local override structure
- distinguish `footer.html` (footer body) from `custom.html` (optional extension hook)

# CSS Override Conflicts

The Stack theme uses multiple CSS layers.

Possible sources of CSS rules:
- theme SCSS
- custom SCSS
- inline styles
- pseudo-elements such as `::before`

When debugging CSS behavior:
- inspect computed styles in browser developer tools
- inspect pseudo-elements too
- avoid assuming a visible line comes from `border-top` without checking actual computed CSS

---

# Prism.js Integration

Prism.js files must exist under `static/`.

Example:
- `static/js/prism.js`
- `static/css/prism.css`

Incorrect placement may cause syntax highlighting to fail.

---

# Redirect Behavior

Legacy FC2 URLs:

```text
/blog-entry-XXXX.html
```

Redirects must convert them to:

```text
/posts/<POST_ID>/
```

Incorrect redirect rules may break old links.

Ensure that both:
- `nikki-san.pages.dev`
- custom domains

behave consistently.

---

# Archive Generation

Archive pages are generated by:
- `rebuild_archives.py`

Common mistakes:
- editing generated archive files manually
- running Hugo without regenerating archive data

If archive pages look incorrect, regenerate archives first.
