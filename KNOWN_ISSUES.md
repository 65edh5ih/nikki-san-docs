# Known Issues — nikki-san

This document lists **known issues and common pitfalls** in the nikki-san blog system.

It helps AI assistants avoid incorrect debugging approaches.

---

# Hugo Build Errors

## JSON Template Rendering Errors

Example error:

unexpected character '<'

Cause:

A JSON template attempted to render HTML.

Typical scenario:

A template renders `.Content` or `.Page.RenderString` inside a JSON output format.

Fix approach:

Check templates under:

layouts/

Look for `.json` templates and ensure that the output is valid JSON.

# Hugo Local vs Production Behavior

Some features behave differently between:

local development
Cloudflare Pages deployment

Example causes:

external APIs
JavaScript-based features
GitHub-based comment systems

When debugging issues that only appear locally, verify whether the feature depends on network access.

# Theme Override Confusion

The Stack theme may contain multiple files with similar names.

For example:

layouts/partials/article/components/details.html

If the same file exists in the theme and in the repository override directory, Hugo uses the override version.

Debugging rule:

Always check whether a file exists under:

layouts/

before modifying the theme.

---

# CSS Override Conflicts

The Stack theme uses multiple CSS layers.

Possible sources of CSS rules:

theme SCSS
custom SCSS
inline styles

When debugging CSS behavior:

inspect computed styles in the browser developer tools.

Avoid assuming which file defines a style.

---

# Prism.js Integration

Prism.js files must exist under:

static/

Example:

static/js/prism.js
static/css/prism.css

Incorrect placement may cause syntax highlighting to fail.

---

# Redirect Behavior

Legacy FC2 URLs:

/blog-entry-XXXX.html

Redirects must convert them to:

/posts/<ID>/

Incorrect redirect rules may break old links.

Ensure that both:

nikki-san.pages.dev
custom domains

behave consistently.

---

# Archive Generation

Archive pages are generated by:

rebuild_archives.py

Common mistakes:

editing generated archive files manually
running Hugo without regenerating archive data

If archive pages look incorrect, regenerate archives first.

---

# Comment System Behavior

Comments are extracted from FC2 export.

Possible issues:

comment count mismatch
comments missing locally

Check:

comment extraction script
generated comment data
template logic

Do not assume the issue is in Hugo before verifying comment data.

---

# Markdown Rendering

The site uses Hugo Goldmark with:

unsafe = true

This allows HTML inside Markdown.

If HTML rendering breaks, check:

markup configuration
shortcodes
HTML syntax in content files.

---

# Deterministic Generation

The project relies on deterministic builds.

Avoid solutions that introduce:

dynamic runtime APIs
client-side content generation that alters page structure.

---

# When Debugging

Before proposing a fix, the AI should verify:

file structure
relevant templates
scripts generating data
build logs

Avoid speculative debugging.
